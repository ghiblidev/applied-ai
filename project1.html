Overview & Problem

The Challenge

Small biotech companies face a critical competitive intelligence gap. Business development teams spend 20+ hours weekly manually researching the competitive landscape—tracking clinical trials, monitoring patent filings, and analyzing scientific publications across dozens of competitors. This manual process is not only time-intensive but also fragmented, often missing critical signals and providing insights too late to inform strategic decisions.

The Solution

This project develops an autonomous AI system that monitors the competitive landscape in migraine therapeutics, tracking clinical trials, patent filings, and scientific publications across 15+ competitors developing both pharmaceutical drugs and medical devices. The platform autonomously collects data from ClinicalTrials.gov, PubMed, and USPTO patent databases, synthesizes findings using local LLM reasoning, and generates executive-ready intelligence reports.

What Makes This Relevant

The system provides more comprehensive and timely competitive insights than manual processes by continuously monitoring multiple data sources, identifying patterns and connections that human analysts might miss, and delivering synthesized intelligence with strategic recommendations. Technical Architecture

Technology Stack

  • Agent Frameworks: LangGraph for state-based orchestration, CrewAI for hierarchical task delegation
  • LLM: Ollama Llama 3.1 (local inference) + API-based LLM for synthesis and reasoning
  • Data Integration: ClinicalTrials.gov API, PubMed E-utilities API, USPTO Patent API
  • Infrastructure: Docker Compose for containerization, PostgreSQL for structured data, Redis for caching
  • API Layer: FastAPI for RESTful endpoints and async processing

Dual-Framework Approach

The project's most distinctive feature is implementing the identical competitive intelligence workflow using two different multi-agent frameworks. This architectural decision serves multiple purposes: demonstrating deep understanding of agent orchestration paradigms, evaluating framework trade-offs in real-world applications, and showcasing the ability to select appropriate tools based on specific requirements.

LangGraph Implementation

Approach: Explicit state machines with conditional routing and iterative refinement cycles

Strengths:

  • Maximum flexibility and control
  • Granular workflow execution
  • Complex conditional logic
  • Detailed observability

Best For: Complex workflows requiring precise control over agent interactions and state transitions

CrewAI Implementation

Approach: Hierarchical agent collaboration with sequential task delegation

Strengths:

  • Faster development time
  • Intuitive agent coordination
  • Clear role definitions
  • Simplified maintenance

Best For: Straightforward workflows where agent roles map cleanly to business processes

Comparison Metrics

Both implementations share common infrastructure and data integrations, enabling direct performance comparison across:

  • Development Time: Time to implement equivalent functionality
  • Code Complexity: Lines of code, cyclomatic complexity, maintainability index
  • Runtime Efficiency: Execution time, memory usage, API call optimization
  • Maintainability: Ease of debugging, testing, and extending functionality

Implementation Highlights

Agent Specialization: Each framework implements specialized agents for data extraction, entity recognition, competitive analysis, and report synthesis. The LangGraph version uses explicit state transitions between agents, while CrewAI leverages role-based task delegation.

Continuous Monitoring: Both systems run scheduled workflows that check for updates across all data sources, identify new filings or publications, and trigger synthesis pipelines when relevant changes are detected.

Local LLM Integration: Using Ollama for local inference reduces API costs and latency while maintaining privacy for sensitive competitive intelligence data. The system falls back to API-based LLMs for complex reasoning tasks that benefit from larger models.

Technical Learnings So Far

The dual-framework approach has already yielded valuable insights. LangGraph's state machine model provides superior control for complex conditional workflows, particularly when dealing with error handling and retry logic in API integrations. CrewAI's hierarchical model enables faster prototyping and more intuitive code organization, making it easier to onboard new contributors. The choice between frameworks ultimately depends on workflow complexity and team preferences rather than absolute performance differences.
